# OCR Sales · HTML prototype

Static, browser-only mockup of the proposed replacement for the AppSheet `OCR-SalesApp-6151159` CRM. Built for stakeholder review before committing to the full Next.js + Supabase build (Phase 1 onwards).

## How to view

Just open `index.html` in a browser. No build step, no server, no dependencies installed locally.

```
open index.html               # macOS
xdg-open index.html            # Linux
start index.html               # Windows
```

If you'd prefer to serve it (some browsers block CDN fonts on `file://`):

```
python3 -m http.server 8000
# then visit http://localhost:8000
```

## What's real vs mocked

**Real:**
- All 19 AppSheet table names, row counts, and key column patterns (probed from the live AppSheet API on 2026-05-03)
- Information architecture (sidebar groups, page hierarchy, link relationships)
- Layout, typography, color system, responsive behaviour
- Sample data structure derived from the actual API responses

**Mocked / fake:**
- Allottee, broker, and staff names — randomised from a list of plausible Indian names
- Aadhaar, PAN, phone numbers, email addresses — fully fake
- Cheque/RTGS reference numbers — random
- KPI totals (₹213 Cr collected, etc.) — illustrative; real numbers come from the Postgres mirror in Phase 2

**Not connected:**
- "Save", "Submit", "Generate PDF" buttons — go through page transitions or `alert()` only; nothing persists
- Search bar at the top — visual only, no live filter
- Filter pills above tables — visual only, no live filter
- Image previews (Aadhaar / PAN / cheque / signed forms) — show placeholder thumbnails

## Design system

- **Display:** Fraunces (variable serif) for page titles, KPI numbers
- **Body:** Geist Sans
- **Mono:** Geist Mono for IDs, amounts, dates, codes
- **Palette:** Deep ink `#0E0E12` for navigation · warm cream `#F6F1E7` canvas · gold `#B8893B` accent · evergreen `#1F3A2E` for premium dark blocks
- **Aesthetic:** Editorial luxury — premium real estate brand identity, NOT generic SaaS dashboard
- All design tokens live in `assets/styles.css` under `:root`

## Responsive behaviour

- ≥1100px: full layout, 4-column KPI grid, side-by-side detail rails
- 820–1100px: 2-column KPI, stacked detail rails, full sidebar
- <820px: hamburger drawer for sidebar, single-column everything, table rows render as card stacks (each cell on its own line with label)
- <480px: single-column KPIs

Try resizing the browser window to see the breakpoints.

`assets/data.js` was generated by anonymising the responses from a live AppSheet REST API probe of all 19 tables on 2026-05-03. The script that generated it (Python, in chat history) reads `/tmp/appsheet_probe/*.json` and outputs JS-ready data. To regenerate from a fresh probe, re-run the snippet in the chat where this prototype was built.

## What this prototype is NOT

- Not a working app. Nothing saves. Nothing fetches.
- Not the final design — this is to align stakeholders on the **direction** before Phase 1 builds the real Next.js + Supabase application against the same schema.
- Not built in React or any framework — vanilla HTML/CSS/JS so it can be hosted anywhere (or just opened from disk) without any runtime.
